# User Stories & Acceptance Criteria
## AI Quiz Game — v1.0

**Phase:** Requirements  
**Agent:** Requirements Agent  
**Date:** 2026-05-31

---

## Epic 1: Room Management

### US-01 — Host creates a game room
**As a** teacher,  
**I want to** create a game room with one click,  
**So that** I can share a code with my students and start a quiz immediately.

**Acceptance Criteria:**
- [ ] Clicking "Create Game" generates a unique 6-character alphanumeric room code
- [ ] Room code is displayed prominently with a copy-to-clipboard button
- [ ] Host is taken to the lobby screen showing the room code and live player count
- [ ] Room code is unique — no two active rooms share the same code
- [ ] Room auto-expires after 2 hours of inactivity

---

### US-02 — Player joins a game room
**As a** student,  
**I want to** join a game by entering a code and my name,  
**So that** I can play without needing to create an account.

**Acceptance Criteria:**
- [ ] Home screen has a "Join Game" input for the room code
- [ ] Invalid or expired codes show a clear error message
- [ ] After valid code entry, player is prompted for a display name (2–20 chars)
- [ ] Player appears in the host's lobby list within 3 seconds of joining
- [ ] Player sees a "Waiting for host to start..." screen in the lobby

---

### US-03 — Host configures the quiz
**As a** teacher,  
**I want to** choose the topic, difficulty, and number of questions,  
**So that** the quiz is appropriate for my class's level.

**Acceptance Criteria:**
- [ ] Host lobby shows selectors for: Topic (5 options), Difficulty (3 levels), Questions (5/10/15)
- [ ] Defaults are: "AI in Daily Life", "Beginner", 10 questions
- [ ] Configuration can be changed any time before clicking "Start Game"
- [ ] At least 2 players must be in the lobby before "Start Game" is enabled

---

## Epic 2: Question Generation

### US-04 — Questions are generated by Claude
**As a** teacher,  
**I want** Claude to generate fresh questions every game,  
**So that** students can't memorise answers from previous sessions.

**Acceptance Criteria:**
- [ ] On "Start Game", Claude API is called with topic + difficulty + count
- [ ] All questions are returned before the first question is shown (no mid-game API calls)
- [ ] Each question has exactly 4 options with exactly 1 correct answer
- [ ] Questions use vocabulary appropriate for the selected grade band
- [ ] If Claude API fails, a pre-seeded fallback question bank is used silently
- [ ] Generation completes in < 5 seconds (p95)

---

### US-05 — Questions include a fun fact explanation
**As a** student,  
**I want to** see a fun explanation after each answer is revealed,  
**So that** I learn something even when I get it wrong.

**Acceptance Criteria:**
- [ ] After answer reveal, a 1–2 sentence "Did you know?" fact is shown
- [ ] The fact is directly related to the correct answer
- [ ] Fact is generated by Claude alongside the question (not a separate API call)
- [ ] Fact uses simple, friendly language appropriate for ages 11–16

---

## Epic 3: Game Flow

### US-06 — Host controls game progression
**As a** teacher,  
**I want to** control when the next question appears,  
**So that** I can pace the game to my class and allow discussion time.

**Acceptance Criteria:**
- [ ] Host sees a "Next Question" button after each answer reveal
- [ ] Players see a "Waiting for host..." message between questions
- [ ] Host can see the question, all options, and the correct answer highlighted
- [ ] Host can end the game early at any point

---

### US-07 — Players answer questions with a timer
**As a** student,  
**I want to** see a countdown and submit my answer quickly,  
**So that** the game feels exciting and fast-paced.

**Acceptance Criteria:**
- [ ] 30-second countdown timer is visible and updates every second
- [ ] Answer options are disabled once submitted or when timer hits 0
- [ ] Submitted answer is highlighted (selected state) immediately on click
- [ ] Correct/incorrect feedback shown after host reveals the answer (not before)
- [ ] Players who don't answer in time receive 0 points for that question

---

### US-08 — Points are awarded based on speed and correctness
**As a** student,  
**I want to** earn more points for answering correctly and quickly,  
**So that** being fast is rewarded.

**Acceptance Criteria:**
- [ ] Correct answer = 100 base points
- [ ] Speed bonus: up to 50 additional points (linear, based on time remaining)
- [ ] Wrong answer or no answer = 0 points
- [ ] Points are calculated server-side (not client-side) to prevent cheating
- [ ] Each player's cumulative score is tracked throughout the game

---

## Epic 4: Leaderboard

### US-09 — Players see live rankings after each question
**As a** student,  
**I want to** see how I rank after each question,  
**So that** I stay motivated to catch up or maintain my lead.

**Acceptance Criteria:**
- [ ] After each answer reveal, a mini leaderboard shows top 5 players + current player's rank
- [ ] Leaderboard shows: rank, display name, score, points gained this round
- [ ] Rankings update correctly after each question
- [ ] "You moved up X places!" indicator shown when player improves rank

---

### US-10 — Final podium screen at game end
**As a** student,  
**I want to** see a celebratory end screen with the top 3 winners,  
**So that** winning feels rewarding and memorable.

**Acceptance Criteria:**
- [ ] Final screen shows a podium with 1st, 2nd, 3rd place players
- [ ] Full ranked leaderboard shown below the podium
- [ ] Each player's total score is displayed
- [ ] "Play Again" button creates a new game with the same settings
- [ ] Confetti or celebration animation plays for the winner

---

## Epic 5: Mobile & Accessibility

### US-11 — Game is playable on a phone
**As a** student,  
**I want to** play on my phone without zooming or scrolling sideways,  
**So that** I don't need a laptop to participate.

**Acceptance Criteria:**
- [ ] All screens are responsive and usable on 375px wide screens (iPhone SE)
- [ ] Answer buttons are large enough to tap comfortably (min 44x44px touch target)
- [ ] Timer and score are always visible without scrolling
- [ ] Text is readable without zooming (min 16px body text)
